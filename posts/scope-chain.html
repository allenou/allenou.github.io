# 作用域

  • var、let、const 之间有什么区别？（2018-06-23）

作用域初始化于上下文创建时，它描述了在上下文中变量的作用范围。作用域可以统分为全局作用域和局部作用域，也可细分为全局作用域、函数作用域、块级作用域。

# 全局作用域

全局作用域是作用域链的最顶级，它可以被全局（任何作用域）访问：

var str = 'Hello world'
function say(){
  console.log(str) // Hello world
}
say()

全局变量并非是处于函数外部的变量，变量的声名方式决定了变量的作用域：

function say(){
  str = 'Hello world'
}
console.log(str) // Hello world

上面的代码中 say 函数中的 str 变量并未使用变量声名关键字声名，如此一来 str 就相当于全局变量，所以可以在函数外部访问到它。

在 for 循环中使用 var 关键字声名的变量其实也是全局变量：

for(var i = 0;i < 10;i++){
  console.log(i)
}

上面的代码相当于：

var i
for(i = 0;i < 10;i++){
  console.log(i)
}

所以可以在 for 外部访问到 i:

for(var i = 0;i < 10;i++){
}
console.log(i)  // 9

# 函数作用域

函数作用域不可在函数外被访问：

function say(){
  var str = 'Hello world'
}
console.log(str) // undefined

父函数也不能访问子函数作用域：

function say(){
  function hello(){
    var str = 'Hello world'
  }
  console.log(str) // undefined
}
say()

# 块级作用域

块级作用域指的是只在所声名改变量的作用域内能被访问，而超出该作用域则无法访问。

上面的 for 循环之所以能在循环体外访问 i 变量是因为 for 循环没有自己的作用域，解决这个为就是通过 let 或 const 关键字声名 i 变量，使得 for 循环内声名的变量只在循环体内有效：

for(let i = 0;i < 10;i++){
}
console.log(i) // i is not defined

块级作用域的出现同时也解决了变量提升问题：

console.log(str) // Hello world
var str = 'Hello world'

上面代码中，使用 var 关键字声名的变量可在声名前访问，而使用 let/const 声名则不会出现这种问题：

console.log(str) // str is not defined
let str = 'Hello world'

# 词法作用域

词法作用域（静态作用域）是在词法分析阶段就已确定：

var str = 'Hello world'
function fn1(){
  console.log(str)
}
function fn2(){
  var str = 'Hello baby'
  fn1()
}
fn2()

按照作用域就近查找原则，上面的代码理应打印 Hello baby，因为 fn1 函数未声名 str 变量，会向父级作用域（fn2）查找，而 fn2 函数中存在 str 变量。但实际打印的是 Hello world,因为 Javascript 遵循的是词法作用域规则。
