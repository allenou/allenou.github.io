# git 账号输错了怎么办

第一次 clone 代码托管平台上的代码时会弹出一个登录框：

如果不小心输错了密码首先会导致 clone 失败，再次执行 clone 命令登录框就不会弹出了。因为账号在第一次输过后就保存系统中了，下次再操作也不会提示登录了，这本来是很人性化的操作，但是如果第一次输错了账号就会变得很尴尬。

解决办法是找到保存这些账号的地方然后修改错误的账号，在 windows 中这些账号保存在 windows 凭据（控制面板=>用户帐户=>凭据管理器=> windows 凭据）的普通凭据中：

找到相应的凭据就可以重新编辑了，当然也可以直接删除，删除后执行 clone 命令时便会重新弹出登录框。
