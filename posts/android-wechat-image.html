# 安卓端微信点击图片被放大问题

最近的抽奖活动没有基于框架开发，在使用同事的安卓手机在微信调试活动页面时，不小心点到了背景图，结果图片被放大了，有点类似于点击公众号文章图片后触发的图片浏览效果:

我有点摸不着头脑，为什么只有安卓端的微信有这种问题，之前基于 Vue 开发的项目从来没出现过这种问题。看来只能阻止图片的默认的默认事件了:

var imgList = document.querySelectorAll('img')
for(var i=0;i<imgList.length;i++){
    imgList[i].addEventListener('tap',function(e){
        e.preventDefault();
    })
}

这样一来是达到目的了，但是做法太粗暴了，还是使用 CSS 比较优雅：

img {
  pointer-events: none;
}

当元素 pointer-events 属性值为 none 时表示，该元素不会成为鼠标事件的 target，若其子元素的 pointer-events 属性值不为 none ，鼠标事件可以对其子元素生效。下面是一个实际列子：

<style>
div{
    display:inline-block;
    padding:20px;
    background-color:blue;
    pointer-events: none; 
}
p{
    width:50px;
    height:50px;
    background-color:red;
}
</style>
 <div onclick="alert('div')">
    <p  onclick="alert('p')"></p>
</div>

div 的 pointer-events 属性值为 none 时，自身以及子元素事件无效。

<style>
div{
    display:inline-block;
    padding:20px;
    background-color:blue;
    pointer-events: none; 
}
p{
    width:50px;
    height:50px;
    background-color:red;
    pointer-events: auto; 
}
</style>
 <div onclick="alert('div')">
    <p  onclick="alert('p')"></p>
</div>

当 div 的子元素 p 的 pointer-events 属性值不为 none 时，子元素事件有效并且会在冒泡阶段触发父元素的事件。
